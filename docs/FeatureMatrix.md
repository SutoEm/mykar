# Feature Matrix

| Feature                    | Phase     | MVP   | User Value               | Notes                         |
| -------------------------- | --------- | ----- | ------------------------ | ----------------------------- |
| Register                   | Phase 4   | Yes   | Private account          | Required before garage        |
| Login                      | Phase 4   | Yes   | Access account           | Required before garage        |
| Forgot password            | Phase 4   | Yes   | Account recovery         | Basic flow first              |
| Profile                    | Phase 4   | Yes   | Account clarity          | Minimal profile only          |
| Garage                     | Phase 5   | Yes   | Vehicle list             | First product surface         |
| Add vehicle                | Phase 5   | Yes   | Start vehicle memory     | Keep form short               |
| Edit vehicle               | Phase 5   | Yes   | Correct information      | Must preserve existing memory |
| Delete vehicle             | Phase 5   | Yes   | Control records          | Confirmation required         |
| Maintenance records        | Phase 6   | Yes   | Preserve service history | Core value                    |
| Document storage           | Phase 6   | Yes   | Preserve important files | Storage rules required        |
| Mileage                    | Phase 6   | Yes   | Context for records      | Manual entry first            |
| Notes                      | Phase 6   | Yes   | Capture informal memory  | Plain text first              |
| Fuel tracking              | Phase 6   | Later | Usage history            | Not required for first MVP    |
| Basic reminders            | Phase 8   | Yes   | Prevent missed actions   | Simple dates first            |
| Missing data prompts       | Phase 8   | Yes   | Improve trust            | Rule-based first              |
| Vehicle memory AI answers  | Phase 7   | Later | Faster understanding     | Must cite stored records      |
| Generic AI chat            | Graveyard | No    | Low early signal         | Do not build first            |
| OBD live data              | Phase 9   | No    | Diagnostics              | Later hardware phase          |
| Marketplace                | Phase 10  | No    | Commerce                 | Later business phase          |
| Service provider workflows | Phase 10  | No    | Partner operations       | Later business phase          |
| Developer portal           | Phase 11  | No    | Ecosystem                | Later platform phase          |

## Matrix Rules

- MVP means required to prove trusted vehicle memory.
- Later means useful, but not required for the first product proof.
- Graveyard means intentionally not built until a later decision review.
