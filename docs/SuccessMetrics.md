# Success Metrics

Phase 1 defines what the product must prove. Implementation phases should use these as product gates.

## Activation

- User creates an account.
- User adds first vehicle.
- User adds at least one memory item.
- User understands what is missing from the vehicle record.

Target for MVP: 60 percent of new users add a vehicle and one memory item in the first session.

## Trust

- User returns to view a vehicle record.
- User uploads or records real maintenance data.
- User corrects or completes missing information.
- User uses stored information to answer a vehicle question.

Target for MVP: 40 percent of activated users add a second memory item within 14 days.

## Retention

- User returns before a maintenance or document deadline.
- User opens a reminder.
- User adds new information after the first week.

Target for MVP: 25 percent of activated users return within 30 days.

## Quality

- Critical flows pass automated checks.
- No high or moderate audit findings in CI.
- Documentation is updated when scope changes.
- Feature work maps to the phase matrix.

## Business Signal

- Users say the vehicle record feels more reliable than their previous method.
- Users can describe what My Kar does in one sentence.
- Users would be disappointed if the garage and memory timeline disappeared.
