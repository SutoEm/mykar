# User Stories

## Phase 4: Authentication

- As a vehicle owner, I want to register so I can keep my vehicle memory private.
- As a vehicle owner, I want to log in so I can access my garage from my phone.
- As a vehicle owner, I want to reset my password so I do not lose access to my records.
- As a vehicle owner, I want to view my profile so I can confirm my account details.

## Phase 5: Vehicle Core

- As a vehicle owner, I want to add a vehicle so I can start building its memory.
- As a vehicle owner, I want to edit vehicle details so the profile stays accurate.
- As a vehicle owner, I want to remove a vehicle so my garage stays clean.
- As a vehicle owner, I want to see all vehicles in a garage so I can choose the right vehicle quickly.

## Phase 6: Vehicle Memory

- As a vehicle owner, I want to add a maintenance record so I can remember what was done.
- As a vehicle owner, I want to upload a document so important files are not lost.
- As a vehicle owner, I want to add mileage so records have useful context.
- As a vehicle owner, I want to add notes so informal knowledge is preserved.
- As a vehicle owner, I want to see a timeline so I can understand the vehicle story.

## Phase 7: AI Layer

- As a vehicle owner, I want to ask when the last maintenance happened so I can avoid searching manually.
- As a vehicle owner, I want to know what is missing from the vehicle memory so I can improve trust.
- As a vehicle owner, I want AI answers to cite stored records so I can verify the answer.

## Acceptance Rules

- Each story must map to one phase.
- A story should not require later-phase infrastructure.
- AI stories must depend on recorded vehicle memory, not generic guessing.
