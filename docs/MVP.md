# MVP

The MVP should prove that a vehicle owner can create and trust a useful vehicle memory.

## MVP Objective

Create a focused flow where a user can register, add a vehicle, record maintenance, store key documents, and receive basic reminders.

## Primary User

The first MVP user is an individual vehicle owner who wants a reliable record of maintenance, documents, and ownership context without managing spreadsheets, paper folders, or scattered photos.

## Core Loop

1. Create account.
2. Add vehicle.
3. Add maintenance, document, mileage, or note.
4. See the vehicle memory timeline.
5. Receive a reminder or missing-data prompt.
6. Improve the vehicle record over time.

## Initial Scope

- Authentication
- Vehicle profile creation
- Maintenance record capture
- Document upload
- Mileage notes
- Basic reminders

## MVP Success Criteria

- A user can create a trusted vehicle profile in less than 3 minutes.
- A user can add a maintenance record without needing a mechanic's terminology.
- A user can find stored documents quickly.
- A user understands what information is missing from the vehicle memory.

## Out of Scope

- Marketplace
- OBD-II integration
- Service provider platform
- Advanced AI automation
- Generic AI chat
- Social features
